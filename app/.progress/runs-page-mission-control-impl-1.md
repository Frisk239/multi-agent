# runs-page-mission-control-impl-1

## 主题
运行页从「运维按钮堆叠 + 宽表重复导航」改为 Mission Control 密表：状态页签 + 单条情境提示 + 行内仅主操作。

## 问题（用户截图）
- 顶栏/横幅多入口重复：看板仅失败、收件箱失败、环境诊断、收尸、运行时、不可用智能体…
- 每行失败列再复制「打开诊断 / 看板仅失败 / 收件箱」
- 行内「筛」链与类型/小队碎链接造成噪声
- 与集合页顶栏收口不一致

## Multica
- 真站无独立 `/runs` 工作区页（404）；本仓超车：本地观测列表

## 实现
- 重写 `RunsPage.tsx`
  - 顶栏：`刷新` + `更多`（看板失败/收件箱/环境/CLI/收尸…）
  - 页签：失败 | 在途 | 全部 | 完成（默认 failed）
  - 筛选折叠；Agent/小队/队长/细分状态
  - insight 条：cwd 时主 CTA「配置 cwd」；否则「看板失败」；在途仅批量取消
  - 表列：状态 · 任务 · Agent · 说明 · 时间 · 操作（时间线/轨迹/再执行）
  - 去掉行内 fail-links 与「筛」
- `globals.css`：`.runs-scope-tabs` / `.runs-insight` / 密表列宽

## 验收（Playwright local）
- header：`刷新` + `更多`
- tabs：失败/在途/全部/完成/筛选
- insight：cwd 文案 + `配置 cwd`
- 列：状态 任务 Agent 说明 时间 操作
- 行样例：无 fail-links；操作=时间线 轨迹 再执行

## 决策
- 运维入口上收（顶栏更多 + 单 insight），不在每行重复
- 默认仍 failed，服务本地运维主场景

## 下一刀建议
- 失败按 code 聚合（cwd/cli/orphan 分组折叠）
- Issue 标题进任务列（需 API 联表或客户端缓存）
- 侧栏「本地运维」标签与知识/观测 IA 是否再命名（截图仍显示本地运维，可能旧缓存）
