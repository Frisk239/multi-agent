# UX P2 Polish Implementation Plan

> 前序：UX Trust A–D · Day-0 E1–E4 已合 main。  
> 本阶段：对标 Multica 后仍值得做的 **P2 打磨**（非 P0 硬洞）。

**默认顺序：**

```
P2-A 看板列表视图 → P2-B 通知偏好 → P2-C Chat 更深流式 → （降级）Wiki 根可配置 · session resume 调研记债
```

| ID | 切片 | 用户可感 |
|---|---|---|
| **P2-A** | Issues `?view=list\|board` | 表格式扫 issue，密度更低 |
| **P2-B** | Inbox 成功推送偏好 | Settings 开关，持久化；默认仍降噪 |
| **P2-C** | Chat `message_delta` → partial | 思考卡 partial 更跟手 |
| **P2-D** | Wiki 根可配置 | `MA_WIKI_DIR` / meta 展示（非 per-project 分库） |
| **P2-E** | session resume | **本阶段不做**；记 progress 债 |

---

## P2-A 列表视图

- URL `view=list|board`（默认 board）
- 顶栏切换；list 用 table：identifier · title · status · assignee · project · updated
- 行点进 `/?issue=` 或现有详情深链

## P2-B 通知偏好

- `~/.multi-agent/inbox-prefs.json`：`{ notifyIssueSuccess: boolean }`
- inbox-writer 读此文件（env `MA_INBOX_NOTIFY_SUCCESS` 仍覆盖为 true）
- Settings 健康区开关 + API GET/PUT

## P2-C Chat 流

- `run:progress` 若像正文则 `appendPartial`（防 log 噪音：排除 `[claude]`/`等待本机` 等）
- 可选：delta 事件单独字段（已有 progress）

## P2-D Wiki 根

- `getWikiDir`：`MA_WIKI_DIR` > workspace/wiki > cwd/wiki
- meta 暴露 `source: env|workspace|cwd`

## Out

- 真 CLI session resume / PriorWorkDir 全套  
- 看板手动排序视图  
- Wiki 按 projectId 多目录大迁移
