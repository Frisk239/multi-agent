# .progress — 跨会话交接文档

每个垂直切片的计划者-执行者交接存在这里。**这是你跨数月、数十会话不丢上下文的记忆层。**

## 命名规则

```
<slice-id>-<role>-<seq>.md

例：
s01-planner-1.md     S01 切片计划者的第一份（设计 + 拆分）
s01-impl-1.md        S01 执行者 1 的交接（实现 + 自测）
s01-impl-2.md        S01 执行者 2 的交接
s01-planner-2.md     S01 计划者验收 + 给下个切片的注意点
```

## 工作流（见 [AGENTS.md](../../AGENTS.md) §工程模式）

```
计划者 → 写 planner handoff（拆分 + 验收标准）
  ↓
执行者 1 → 实现 → 写 impl handoff（完成 + 自测 + 偏离 + 遗留）
  ↓
计划者 → 验收 → 写给执行者 2 的注意点
  ↓
执行者 2 → 读注意点 → 实现 → 写 impl handoff
  ↓
... 顺序执行 ...
  ↓
最后一个执行者 → 跑通验收
  ↓
计划者 → 写切片总结 → 人在 main 合并
```

## 规则

- **每个会话开始前先读本目录最新的 handoff**
- 模板：[`_TEMPLATE.md`](_TEMPLATE.md)
- 执行者**必须**写自测结果（贴命令输出），不能只说"通过了"
- 计划者**必须**写验收结论 + 给下个执行者的注意点（不是新计划）
- 偏离计划必须记录原因——这是后面调试和论文写作的素材
